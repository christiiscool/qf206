QF206 Quant Trading Project
===========================

This project implements the current research pipeline for dynamic factor timing on a fixed 6-stock demo universe:

- Universe: `AAPL`, `MSFT`, `NVDA`, `JPM`, `XOM`, `UNH`
- Rebalance: monthly
- Default production model: ElasticNet factor-timing regression
- Comparison models: Ridge, ElasticNet, XGBoost
- Options component: rule-based SPY options warning overlay that scales portfolio exposure via `lambda_t`

## Project Structure

- `src/`
  - `config.py`: Central configuration (paths, dates, hyperparameters, portfolio and overlay settings).
  - `universe.py`: Universe helpers and metadata.
  - `data_download.py`: yfinance-based OHLCV downloads for the demo tickers and SPY.
  - `features.py`: Factor feature engineering and aggregation to a single monthly panel dataset.
  - `labels.py`: Forward one-month return labels and helper columns.
  - `factor_portfolios.py`: Cross-sectional factor signal and factor return construction.
  - `factor_timing_model.py`: Regime feature engineering plus Ridge, ElasticNet, and XGBoost factor-timing regressors.
  - `portfolio_allocator.py`: Factor allocation and stock-weight combination logic.
  - `backtest.py`: Walk-forward backtest with transaction costs and options-risk exposure scaling.
  - `options_signal_data.py`: WRDS options data cleaning and monthly aggregation.
  - `options_warning_signals.py`: Rule-based monthly warning score and `lambda_t` generation.
  - `evaluation.py`: Backtest summary statistics and exported metrics.
  - `plots.py`: Plot generation for equity, drawdown, turnover, weights, and overlay diagnostics.
- `data/raw/`: Raw price and source files.
- `data/processed/`: Processed monthly panel datasets.
- `outputs/`: Default-model outputs plus optional comparison subfolders.
- `run_pipeline.py`: Default production pipeline. Runs ElasticNet unless `--compare-models` is passed.
- `run_model_comparison.py`: Explicit comparison runner for Ridge, ElasticNet, and XGBoost.

## Dependencies

Key Python libraries:

- `pandas`, `numpy`, `scipy`
- `scikit-learn`
- `xgboost`
- `yfinance`
- `matplotlib`, `seaborn`
- `torch` (optional, for the multitask MLP)

Install them into your environment:

```bash
cd qf206_quant
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Data Sources

- **OHLCV prices**: `yfinance` downloads for the configured demo tickers and the index proxy `SPY`.
- **Options signals**: WRDS-style options export configured through `src/config.py` and consumed by the warning-signal pipeline.

`data/raw/data_sources.json` is generated by the pipeline to record the concrete URLs and timestamps used to build each dataset.

## Running the Full Pipeline

Once dependencies are installed:

```bash
cd qf206_quant
python run_pipeline.py
```

This will:

1. Download daily OHLCV for the configured demo tickers and SPY.
3. Construct a single **monthly panel** with rows `(month_end, ticker)` and columns `features + labels`.
4. Train the default factor-timing model with expanding-window validation.
5. Run the monthly walk-forward backtest with transaction costs.
6. Apply the options warning overlay by scaling exposure when the warning score breaches its threshold.
7. Save processed data, summary tables, and plots under `data/processed/` and `outputs/`.

To run comparison mode instead:

```bash
python run_pipeline.py --compare-models
```

Comparison mode writes per-model outputs under `outputs/ridge/`, `outputs/elasticnet/`, and `outputs/xgboost/`, plus a summary table at `outputs/model_comparison.csv`.

## Configuration

Most behavior is controlled via `src/config.py`:

- Demo universe and dates
- Feature lookback windows
- Factor-timing model choice and hyperparameters
- Portfolio construction and transaction cost settings
- Options warning overlay parameters

You can either edit the defaults in `config.py` or extend it to read from a YAML/JSON file.
